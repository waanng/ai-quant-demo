1. 项目概述

基于已完成的海龟交易策略代码，开发一个交互式 Web 演示看板，支持实时调整标的、时段、策略参数，并即时呈现回测结果与可视化。

1.1 设计目标

目标 说明
交互性 所有参数均可通过侧边栏实时调整，点击运行即出结果
即时反馈 参数修改后一键重算，无需重启
可视化丰富 主图、资金曲线、指标卡片、交易明细一屏展示
可移植 本地一键启动，也可部署到服务器远程访问

1.2 技术选型

组件 选型 理由
框架 Streamlit Python 原生，无需前端知识，适合量化看板
图表 Plotly（交互式）+ Matplotlib（静态导出） Plotly 支持缩放/悬浮，Matplotlib 用于 PNG 导出
数据 Pandas 已有 CSV 数据，直接加载
样式 Streamlit 原生 + 自定义 CSS 轻量美化，不引入额外 UI 框架
部署 本地 streamlit run 无需额外服务器

备选方案：如果倾向 Jupyter Notebook 内交互，可用 ipywidgets + ipyplots，但交互体验不如 Streamlit。

2.2 数据字段

ts_code, trade_date, open, high, low, close, pre_close, change, pct_chg, vol, amount

2.3 数据加载逻辑

  • 启动时自动扫描项目目录下所有 *行情数据.csv 文件
  • 从文件名解析标的名称（如 寒武纪行情数据.csv → 寒武纪）
  • 读取 ts_code 列获取股票代码和板块信息
  • 根据 ts_code 后缀自动判断最小交易单位：
  • .SH 且代码以 688 开头 → 科创板，200 股/手
  • .SZ 且代码以 300 开头 → 创业板，100 股/手
  • 其余 → 100 股/手
  • 统一日期格式为 YYYY-MM-DD
  • 加载后缓存到 st.session_state，避免重复读取

扩展性：如果后续新增标的数据 CSV，看板自动识别并出现在下拉菜单中，无需改代码。

5. 策略引擎

5.1 复用已有代码

看板直接复用海龟策略的核心函数，确保逻辑一致：

函数 输入 输出

5.3 运行流程

用户调整参数 → 点击"运行回测"
      │
      ▼
  读取数据 → 按时段筛选
      │
      ▼
  计算通道 + ATR → 生成信号
      │
      ▼
  执行回测（含加仓、止损、费用）
      │
      ▼
  计算指标 → 更新图表和表格
      │
      ▼
  渲染到主区域

5.4 性能考虑

  • 数据加载后缓存在 st.session_state，切换时段不重新读 CSV
  • 回测计算用 NumPy 向量化，单次 < 1 秒
  • Plotly 图表用 WebGl 渲染，728 个数据点流畅
  • 如果数据量超过 5000 行，自动降采样显示（不影响回测精度）
