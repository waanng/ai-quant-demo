股票数据机器学习分类模型示例¶

本 Notebook 使用 model_data_stock .csv 建立二分类模型，并比较逻辑回归、决策树、随机森林与 K 近邻（KNN）的样本外表现。

数据严格按照日期划分：前三个季度为训练集，2022-03-31 为验证集，2022-06-30 为测试集。这样可以避免随机切分把未来数据泄漏到训练阶段。

注意：当前数据文件没有提供 Y 的业务定义，因此本文只把 Y=True 称为“正类”，不将其解释为上涨、盈利或投资信号。

2. 按日期设置训练集、验证集和测试集¶

  • 训练集：2021-06-30、2021-09-30、2021-12-31
  • 验证集：2022-03-31
  • 测试集：2022-06-30

Date、Code 是标识字段，不作为模型输入。所有填补和标准化参数仅在训练集上拟合。

valid_date = pd.Timestamp('2022-03-31')
test_date = pd.Timestamp('2022-06-30')

train_df = df[df['Date'].isin(train_dates)].copy()
valid_df = df[df['Date'].eq(valid_date)].copy()
test_df = df[df['Date'].eq(test_date)].copy()

if min(len(train_df), len(valid_df), len(test_df)) == 0:
    raise ValueError('至少一个数据集为空，请检查日期设置。')
if not (train_df.Date.max() < valid_df.Date.min() < test_df.Date.min()):
    raise ValueError('时间顺序不正确，存在潜在数据泄漏。')

feature_cols = [c for c in df.columns if c not in ['Date', 'Code', 'Y']]

3. 构建并训练四种分类模型¶

逻辑回归和 KNN 对变量尺度敏感，因此使用中位数填补和标准化；决策树和随机森林仅进行中位数填补。这里使用固定的教学型参数，后续可只利用训练集和验证集进行调参。

In [4]:
scaled_preprocess = [
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
]
tree_preprocess = [('imputer', SimpleImputer(strategy='median'))]

models = {
    'Logistic Regression': Pipeline(scaled_preprocess + [
        ('model', LogisticRegression(max_iter=2000, class_weight='balanced', random_state=RANDOM_STATE))
    ]),
    'Decision Tree': Pipeline(tree_preprocess + [
        ('model', DecisionTreeClassifier(
            max_depth=6, min_samples_leaf=30, class_weight='balanced',
            random_state=RANDOM_STATE
        ))
    ]),
    'Random Forest': Pipeline(tree_preprocess + [
        ('model', RandomForestClassifier(
            n_estimators=300, max_depth=10, min_samples_leaf=10,
            max_features='sqrt', class_weight='balanced',
            random_state=RANDOM_STATE, n_jobs=-1
        ))
    ]),
    'KNN': Pipeline(scaled_preprocess + [
        ('model', KNeighborsClassifier(n_neighbors=25, weights='distance', n_jobs=-1))
    ])
}

fitted_models = {}
for name, model in models.items():
    print(f'正在训练：{name}')
    fitted_models[name] = clone(model).fit(X_train, y_train)
print('四个模型训练完成。')

4. 模型评估与 AUC¶

AUC 是 ROC 曲线下的面积，衡量模型将正类排在负类之前的能力。AUC=0.5 接近随机排序，越接近 1 越好。这里同时报告 Accuracy、Precision、Recall 和 F1，避免只看单一指标。

In [5]:
def evaluate_models(fitted, X, y, dataset_name):
    rows = []
    probabilities = {}
    for name, model in fitted.items():
        prob = model.predict_proba(X)[:, 1]
        pred = (prob >= 0.5).astype(int)
        probabilities[name] = prob
        rows.append({
            '数据集': dataset_name,
            '模型': name,
            'AUC': roc_auc_score(y, prob),
            'Accuracy': accuracy_score(y, pred),
            'Precision': precision_score(y, pred, zero_division=0),
            'Recall': recall_score(y, pred, zero_division=0),
            'F1': f1_score(y, pred, zero_division=0),
        })
    return pd.DataFrame(rows), probabilities

valid_metrics, valid_probabilities = evaluate_models(
    fitted_models, X_valid, y_valid, 'Validation (2022-03-31)'
)
test_metrics, test_probabilities = evaluate_models(
    fitted_models, X_test, y_test, 'Test (2022-06-30)'
)
metrics = pd.concat([valid_metrics, test_metrics], ignore_index=True)
display(metrics.round({
    'AUC': 4, 'Accuracy': 4, 'Precision': 4, 'Recall': 4, 'F1': 4
}))

5. 绘制测试集 ROC 曲线¶

ROC 曲线横轴是假正率（FPR），纵轴是真正率（TPR）。灰色对角线表示随机分类器。曲线越靠近左上角、AUC 越高，模型的整体排序能力越强。

7. 使用说明与局限性¶

  1. 验证集用于比较模型和调整参数；测试集只用于最终样本外评价。
  2. 本数据只有 5 个季度截面，测试结果可能对市场阶段非常敏感。
  3. 不同季度的正类比例变化明显，因此 AUC 应与 Precision、Recall、F1 和混淆矩阵共同判断。
  4. 在用于投资研究前，必须确认 Y 的定义、预测期限，以及所有财务特征在当时是否已经公开，排除前视偏差。
  5. 本示例没有考虑交易成本、滑点、停牌、涨跌停和组合构建，因此不能直接作为交易策略。
